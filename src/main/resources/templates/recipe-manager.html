Krótki opis:

Short description:

Myśl, aby napisać aplikację internetową do przechowywania przepisów kuchennych zrodziła się w mojej głowie, gdy byłem w sklepie spożywczym. Nie mając domowej książki kuchennej pod ręką, potrzebowałem sprawdzić co dokładnie potrzebuję do upieczenia ciasta. Z czasem wykiełkowała ona do pomysłu, aby napisać projekt z wykorzystaniem technologii, które już znam, oraz z tymi z którymi jeszcze nie było dane mi się zetknąć. Projekt oparłem o najlepiej znaną mi Javę oraz framework Spring. Do zarządzania schematami bazy danych wykorzystałem Liquibase, a do mapowania ich w kodzie skorzystałem z frameworku Hibernate. Backend aplikacji połączyłem z frontendem za pomocą Thymeleafa. Do wygodnego pisania strony internetowej skorzystałem z nowego dla mnie Bootstrapa. Na koniec projekt połączyłem z platformą dostarczania wiadomości email (Mailtrap), co wymagało ode mnie zapoznania się z dokumentacją ich API. Projekt napisałem w celu utrwalenia wiedzy i wystawienia się na nowe wyzwania, aby poszerzyć swoje kompetencje.

The thought of writing a web application to store kitchen recipes arose in my mind when I was at the grocery store. With no home cookbook on hand, I needed to see exactly what I needed to bake a cake. Over time, it germinated into an idea to write a project using technologies I was already familiar with and those I hadn't yet encountered. I based the project on the Java I was most familiar with and the Spring framework. To manage the database schemas I used Liquibase, and to map them in the code I used the Hibernate framework. I connected the backend of the application to the frontend using Thymeleaf. I used Bootstrap, which is new to me, to conveniently write the website. Finally, I connected the project to an email delivery platform (Mailtrap), which required me to read their API documentation. I wrote the project to consolidate my knowledge and expose myself to new challenges to broaden my competence.

Jest to druga odsłona aplikacji edukacyjnej, którą stworzyłem do nauki nazw krajów. Zdecydowałem się ją stworzyć, ponieważ nie mogłem znaleźć w Internecie gry, która spełniałaby moje oczekiwania dotyczące listy krajów. Gra zawiera kraje uznawane na całym świecie, ale stworzyłem osobną kategorię z krajami, które są uznawane tylko częściowo. Mapa nie odzwierciedla moich przekonań politycznych. W projekcie wykorzystałem technologię JavaFX do stworzenia aplikacji okienkowej. Jako urozmaicenie nauki i próbę zachęty, dodałem system punktów, który zależy od liczby odgadniętych krajów i czasu rozgrywki. Dla graczy lubiących rywalizację, stworzona została również lista z 10 najlepszymi, która jest serializowana przed zamknięciem aplikacji, aby nie stracić wyników po jej wyłączeniu.

This is the second installment of an educational app I created for learning the names of countries. I decided to create it because I could not find a game on the Internet that met my expectations for a list of countries. The game includes countries that are recognized around the world, but I created a separate category with countries that are only partially recognized. The map does not reflect my political beliefs. In the project, I used JavaFX technology to create a windowed application. As a learning enrichment and an attempt at encouragement, I added a point system that depends on the number of countries guessed and gameplay time. For players who like competition, a list with the top 10 was also created, which is serialized before the application is closed so that scores are not lost when the application is shut down.

O projekcie:

About the project:

Kolorystyka projektu jest bardzo prosta czarno-biała, ponieważ pisząc ten projekt frontend nie był moim priorytetem. Skupiłem się w nim na backendzie, oraz zaprezentowaniu moich umiejętności i znajomości technologii programowania aplikacji webowej. Na stronie głównej prezentuję umiejętności w rozpoznawaniu zalogowanego użytkownika oraz zarządzanie templatami Thymeleafa. W zależności od zalogowania dla użytkownika albo wyświetla się strona główna opisująca projekt (Lorem Ipsum…) lub strona wyświetlające karty z wprowadzonymi do bazy danych przepisami użytkownika.

The color scheme of the project is very simple black and white, because when writing this project the frontend was not my priority. In it I focused on the backend, and showcasing my skills and knowledge of web application programming technologies. On the frontend, I demonstrate my skills in recognizing a logged-in user and managing Thymeleaf templates. Depending on the login for the user, either a home page describing the project (Lorem Ipsum...) or a page displaying tabs with the user's recipes entered into the database is displayed.

Strony z logowaniem i rejestracją reprezentują moje umiejętności z operowaniem danych wrażliwych oraz w jaki sposób są przechowywane, a także przesyłane pomiędzy użytkownikiem, a serwerem. Przy operowaniu na encjach wykorzystuje odpowiednie klasy DTO, aby zabezpieczyć aplikację przed wyciekiem danych. W każdym endpoincie stosuję klasę DTO mapującą tylko te pola w encji, które są wymagane.

The login and registration pages represent my skills with handling sensitive data and how it is stored and also transferred between the user and the server. When operating on entities, I use appropriate DTO classes to protect the application from data leakage. In each endpoint, I use a DTO class that maps only those fields in the entity that are required.

Strona z dodawaniem przepisów jej flow reprezentuje ponownie moją znajomość Thymeleafa. Trudność z jaką się spotkałem na tej podstronie była nieprzewidywalność liczby składników w przepisie. Musiałem poznać sposób w jaki sposób Thymeleaf interpretuje oraz zapisuje listy na stronie, oraz sposób połączenia tego z kontrolerem w Springu. Udało mi się to ostatecznie wprowadzając do modelu endpointa klasy IngredientsListWrapperDto zawierającą w sobie listę składników i połączenia jej już w klasie service z przepisem. Przydała się też znajomość JavaScripta, w którym operuję dodawanie pól do formularza na stronie.

The page with the addition of recipes her flow represents again my knowledge of Thymeleaf. The difficulty I encountered on this subpage was the unpredictability of the number of ingredients in a recipe. I had to learn how Thymeleaf interprets and saves lists on the page, and how to connect this to the controller in Spring. I finally succeeded by introducing the IngredientsListWrapperDto class containing the list of ingredients in the endpoint model and linking it already in the service class to the recipe. It also came in handy to know JavaScript, where I operate adding fields to the form on the page.
